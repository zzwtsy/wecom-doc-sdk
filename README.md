# 企业微信文档 SDK

实现企业微信 `docs\企业内部开发\服务端API\文档\管理智能表格内容` 相关 api 接口的 python 封装

要求：
    1. python 代码要有良好的中文注释
    2. http 请求使用 httpx 库
    3. 类型模型使用 pydantic
    4. 代码需要有良好的类型支持
    5. 代码需要保留后续其他接口的扩展空间

## 开发规范（摘要）

- 开发环境：Python 3.10，建议使用 `uv`/虚拟环境
- 工具链基线：`ruff` + `black` + `pytest`
- 代码规范：中文注释、完整类型标注、公开 API 需 docstring
- 结构约束：接口实现按模块拆分，避免单文件堆叠
- Git 约定：分支命名、提交信息与版本发布规范

详细规范见：`CONTRIBUTING.md`
